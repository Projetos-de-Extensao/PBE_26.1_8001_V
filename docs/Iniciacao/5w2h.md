## Introdução

<p align="justify">
    O 5W2H é uma ferramenta de gestão estratégica amplamente utilizada no planejamento 
    e na execução de planos de ação. Estruturada em sete perguntas fundamentais 
    (What?, Why?, Where?, Who?, When?, How? e How much?), a metodologia tem como 
    objetivo esclarecer, organizar e comunicar ideias de forma objetiva e sistemática. 
    Neste documento, o 5W2H é aplicado em duas perspectivas complementares — a Visão 
    Geral do projeto e a Visão do Produto — para definir as diretrizes do Sistema de 
    Gestão e Validação Automática de Estágios do Ibmec, servindo como base para alinhar 
    objetivos, responsabilidades e decisões ao longo do desenvolvimento.
</p>

## Metodologia

<p align="justify">
    A equipe realizou reuniões de alinhamento para aprofundar a compreensão sobre o 
    escopo e os requisitos do projeto. A partir das discussões conduzidas, foi possível 
    estruturar e responder às sete questões do 5W2H de forma consistente e fundamentada, 
    contemplando tanto a perspectiva organizacional quanto a perspectiva técnica do produto.
</p>

## Questões 5W2H

### Versão 1.0

### Visão Geral

| **Questão**                         | **Resposta**                                                                                                                                              |
| ----------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What?** (O que será feito?)       | Desenvolvimento de uma API Backend para gestão e validação automática de propostas de estágio, com regras customizáveis por curso.                        |
| **Why?** (Por que isso será feito?) | Para automatizar a conformidade com a Lei 11.788/08, reduzir a carga operacional dos coordenadores e acelerar o processo de avaliação das propostas.      |
| **Where?** (Onde será feito?)       | Desenvolvimento via GitHub Codespaces, com documentação publicada por meio do MkDocs via GitHub Pages.                                                    |
| **When?** (Quando será feito?)      | Ao longo do primeiro semestre de 2026, por meio de iterações incrementais seguindo a metodologia UP/RUP.                                                  |
| **Who?** (Quem irá fazer?)          | Equipe de desenvolvimento composta pelos alunos da disciplina Projeto Back-end.                                                                           |
| **How?** (Como será feito?)         | Utilizando Python e Django para a lógica de negócio, com controle de versão via Git e documentação estruturada em MkDocs.                                 |
| **How much?** (Quanto custará?)     | Aproximadamente 20h semanais de esforço coletivo da equipe, com custo financeiro zero em virtude da adoção exclusiva de tecnologias de código aberto.     |

---

### Versão 2.0

### Visão Produto

| **Questão**                         | **Resposta**                                                                                                                                                                                                                                 |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What?** (O que será feito?)       | Uma API RESTful que recebe os dados dos contratos de estágio e aplica um motor de regras para validar automaticamente a conformidade com a Lei 11.788/08 e as diretrizes internas de cada curso do Ibmec.                                    |
| **Why?** (Por que isso será feito?) | Para reduzir o volume operacional dos coordenadores, diminuir o tempo de espera dos alunos pela aprovação e minimizar erros humanos na conferência das regras acadêmicas e legais.                                                            |
| **Where?** (Onde será feito?)       | A API será desenvolvida e versionada no GitHub, documentada via MkDocs e poderá ser hospedada em ambiente de nuvem para garantir acessibilidade contínua.                                                                                    |
| **When?** (Quando será feito?)      | O sistema estará disponível para validar propostas em tempo real no momento exato em que o aluno submeter o formulário ou documento no portal institucional.                                                                                  |
| **Who?** (Quem irá fazer?)          | **Usuários diretos:** O sistema institucional do Ibmec. **Usuários indiretos:** Alunos (que submetem os documentos) e coordenadores ou equipe administrativa (que recebem os relatórios de validação).                                       |
| **How?** (Como será feito?)         | Por meio de endpoints que recebem dados de estágio em formato JSON, processam as informações contra as regras de negócio configuradas no banco de dados e retornam um status (Aprovado/Reprovado) com os motivos detalhados das pendências.   |
| **How much?** (Quanto custará?)     | Possíveis custos futuros relacionados à hospedagem em nuvem e manutenção do banco de dados; até o presente momento, o custo financeiro é zero.                                                                                               |

## Conclusão

<p align="justify">
    A elaboração do 5W2H em suas duas dimensões — Visão Geral e Visão do Produto — 
    permitiu à equipe consolidar uma compreensão clara, estruturada e abrangente sobre 
    o Sistema de Gestão e Validação Automática de Estágios do Ibmec. Ao responder às 
    sete perguntas sob perspectivas complementares, tornaram-se evidentes tanto a 
    viabilidade técnica e organizacional do projeto quanto o impacto real que o produto 
    proporcionará ao ambiente institucional.
</p>
<p align="justify">
    Do ponto de vista do projeto, o desenvolvimento mostra-se bem estruturado, com 
    tecnologias de código aberto que garantem custo zero e práticas modernas que 
    asseguram qualidade na entrega. Do ponto de vista do produto, a API propõe resolver 
    uma demanda administrativa crítica, automatizando a verificação de conformidade com 
    a Lei 11.788/08 e reduzindo significativamente a carga manual dos coordenadores. 
    Com esse alinhamento inicial estabelecido, a equipe possui uma base sólida para 
    avançar com clareza nas próximas etapas do desenvolvimento.
</p>
